Emacs key-chord binding tip

Oct 18, 2015

One of the commonly suggested solutions to boost editing productivity in Emacs is usage of the key-chord plugin. I think it is great advice, but once you start to use it the question arises:

Which key combinations should I use as key chords?

I would like to share a productivity tip for those who are looking fo an anwser to this question. So to start let us define what makes a good key chord:

  • easy to remember
  • hard to press by accident
  • pressed using strong fingers

The solution I came up which satisfies the above checklist is to use the digits 3 or 9 as first letters of the keychord and use a letter on the other side of the keyboard as the second key. An example should clarify what I mean: 

(key-chord-define-global "3k" 'kill-buffer)

Easy to remember

The only thing that needs to be remembered is the single letter, the digit is implied by the letter. k is for kill-buffer, k is on the right side of keybord so I press it with 3 - simple.

Hard to press by accident

I have never pressed those by accident while writing prose nor code. Think about it - how often do you type a digit and a letter consecutively?

Pressed using strong fingers

Both 3 and 9 are pressed easily using the middle finger with little movement. The choice of the other key is up to you - but there is plenty to choose from, 3k, 3j, 9f, 9e to name a few.

Other examples with mnemonics: 

(key-chord-define-global "9f" 'projectile-find-file) ; F == File
(key-chord-define-global "9w" 'save-buffer) ; W == Write
(key-chord-define-global "3j" 'avy-goto-subword-1) ; J == Jump
(key-chord-define-global "9r" 'anzu-query-replace-regexp) ; R == Replace
(key-chord-define-global "9t" 'delete-trailing-whitespace) ; T == Trailing
(key-chord-define-global "3u" 'undo-tree-visualize) ; U == Undo
